Dependencies

This bundle requires a number of other symfony bundles that are normally used in dev-environments to be configured. They provide the functionality that the generated code leverages. The bundles will not be pulled into production since this bundle should be added under the require-dev part of your composer.json file.

It is recommended that you add some of them to your composer's require section. Please read on for an explanation of where a dependency should be added, what features each dependency brings and working configuration examples.

Each bundle will have to be enabled in the appropriate section of your app/AppKernel.php file.

At a glance:

  • FriendsOfSymfony/FOSRestBundle
  • nelmio/NelmioApiDocBundle
  • liip/LiipFunctionalTestBundle

FOSRestBundle

Must be added to your composer.json require section

Add to composer:

$ composer require friendsofsymfony/rest-bundle ~1.5

Configuration instructions in their repo: docs

FosRestBundle is used by all the controllers generated by this bundle.

All generated controllers inherit from this bundle.

Example Configuration:

# app/config/config.yml
fos_rest:
    routing_loader:
        default_format: json
    param_fetcher_listener: true
    body_listener: true
    body_converter:
        enabled: true
    format_listener: true
    access_denied_listener:
        json: true
        xml: true
    view:
        view_response_listener: force
        formats:
            xml: true
            json : true
            html : true
        templating_formats:
            html: true
        force_redirects:
            html: true
        failed_validation: HTTP_BAD_REQUEST
        default_engine: twig
    exception:
        codes:
            'Symfony\Component\Routing\Exception\ResourceNotFoundException': 404
            'Symfony\Component\HttpKernel\Exception\ServiceUnavailableHttpException': 500

NelmioApiDocBundle

Must be added to your composer.json require section

Add to composer:

$ composer require nelmio/api-doc-bundle ~2.7

Configuration instructions in their repo docs.

NelmioApiDocBundle generates documentation for your API through the usage of the @Api annotation.

It is used any time the --document flag is used in the controller generator.

Example Configuration:

# app/config/config.yml
nelmio_api_doc:
    name: Excalibur API
    sandbox:
        body_format:
            formats:
                - form
                - json
            default_format: json
        request_format:
            formats:
                json: application/json
                xml: application/xml
            method: format_param
            default_format: json

LiipFunctionalTestBundle

Automatically pulled in to dev and test environments

Configuration instructions in their repo

This bundle is used in the generation of tests for your entities.

Configuration instructions in their repo docs. Example Configuration

# app/config/config_test.yml
liip_functional_test: ~
